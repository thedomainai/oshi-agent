# オシエージェント - AI推し活マネージャー

## プロジェクト概要

推しを登録するだけで、AIが24時間自律的に情報を収集・重要度判定・通知し、遠征計画や予算管理まで行うパーソナルマネージャーサービス。**最も「Agentic AI」らしい設計**（AIが自律的に24時間稼働する）が特徴。

## ハッカソン情報

- **大会**: 第4回 Agentic AI Hackathon with Google Cloud
- **締切**: 2026年2月15日
- **審査基準**: 課題の新規性 / 解決策の有効性 / 実装品質と拡張性
- **技術要件**:

  - Vertex AI / Gemini API / ADK 等のAI技術を1つ以上使用必須

## 解決する課題

### 誰の、どんな課題か

推し活をしている約1,384万人（前年+250万人で急増中）が、X・Instagram・YouTube・公式サイト・ファンクラブなど**散在する情報を追いきれず疲弊**している。

| 困りごと | 具体例 |
|---------|--------|
| 情報が散在 | X、Instagram、YouTube、公式サイト、FC…全部チェックしきれない |
| 重要情報を見逃す | チケット先行販売の告知を見逃してしまう |
| 遠征の計画が大変 | ライブ会場の交通・宿泊を毎回調べる |
| 予算管理ができない | 推し活にいくら使っているか把握できない |

### キラーフレーズ

> **「推しの情報、もう追いかけなくていい。AIが届けます。」**

### 市場規模

| 指標 | 数値 |
|------|------|
| 推し活人口 | **約1,384万人** |
| 市場規模 | **約3兆5,000億円** |
| 1人あたり年間支出 | 平均 **約25万5,000円** |

## 技術アーキテクチャ

### マルチエージェント構成（ADK）

```
┌─────────────────────────────────────────┐
│           Root Agent (Orchestrator)       │
│       全体フローの制御・スケジューリング      │
└──────┬──────┬──────┬──────┬──────────────┘
       │      │      │      │
  ┌────▼──┐┌──▼───┐┌─▼────┐┌▼─────────┐
  │Scout  ││Prior-││Calen-││Trip       │
  │Agent  ││ity   ││dar   ││Agent      │
  │       ││Agent ││Agent ││           │
  │情報    ││重要度 ││予定   ││遠征       │
  │自律収集 ││判定   ││管理   ││計画       │
  └────────┘└──────┘└──────┘└───────────┘
                                ┌──────┐
                                │Budget│
                                │Agent │
                                │予算   │
                                │管理   │
                                └──────┘
```

| エージェント | 役割 | 技術 |
|------------|------|------|
| **Root Agent** | 全体制御、スケジューリング | ADK Orchestrator |
| **Scout Agent** | 推しの公式・SNS情報を自律巡回・収集 | Gemini + Web検索API |
| **Priority Agent** | 収集情報の重要度を判定（チケット発売>日常投稿） | Gemini (分類) |
| **Calendar Agent** | イベント・発売日をカレンダーに自動登録 | Google Calendar API |
| **Trip Agent** | ライブ遠征の交通・宿泊プランを自動作成 | Gemini + Maps API |
| **Budget Agent** | 推し活の支出記録・予算管理 | Gemini (分析) |

### 技術スタック

| レイヤー | 技術 | 用途 |
|---------|------|------|
| フロントエンド | Next.js (React) | WebアプリUI |
| バックエンド | Python + ADK | マルチエージェント制御 |
| AI | Gemini 2.0 Flash | 情報分析・分類・計画生成 |
| 情報収集 | Google Search API / Custom Search | Web情報の自律収集 |
| スケジュール | Google Calendar API | 予定の自動登録 |
| 地図 | Google Maps API | 遠征ルート計算 |
| データストア | Firestore | ユーザーデータ、収集情報 |
| 定期実行 | Cloud Scheduler | Scout Agentの定期巡回 |
| デプロイ | Cloud Run | コンテナデプロイ |

## MVPスコープ（5日間）

### 含めるもの
- 推しの登録（アーティスト名 or URL）
- 情報の自律収集（Web検索ベース）
- 重要度判定・分類
- 収集情報のタイムライン表示
- イベント情報のカレンダー登録
- 遠征プランの自動生成（1件分のデモ）

### 含めないもの（拡張機能）
- リアルタイムSNSモニタリング（APIレートリミット制約）
- プッシュ通知
- グッズ在庫管理
- ファンコミュニティ機能
- 複数推し対応

## 3分デモシナリオ

```
Step 1: 推しを登録（例: あるアーティスト名を入力）
Step 2: AIが自律的に情報収集を開始（バックグラウンド）
Step 3: 収集された情報がタイムラインに表示
  🔴 緊急「ファンクラブ先行チケット受付開始！締切: 明後日18:00」
  🟡 注目「新アルバム発売日決定: 4月23日」
  🟢 日常「本人がInstagramストーリーを更新しました」
Step 4: イベントがカレンダーに自動登録済み
Step 5: ライブの遠征プランを自動生成
  「新幹線 大阪→東京 13:00発、宿泊: 水道橋¥8,000、推定費用: ¥48,000」
Step 6: 月次の推し活レポート
  「今月の推し活支出: ¥35,000（予算の87%）」
```

## 5日間実装計画

| Day | タスク |
|-----|--------|
| Day 1 | ADKセットアップ、Root Agent + Scout Agent（Google Search API連携） |
| Day 2 | Priority Agent + Calendar Agent（Google Calendar API連携） |
| Day 3 | Trip Agent（Maps API連携）+ Budget Agent、フロントエンドUI |
| Day 4 | Cloud Scheduler連携、統合テスト、Cloud Runデプロイ |
| Day 5 | Zenn記事執筆、YouTube動画撮影、最終調整 |

## 実装上のリスクと対策

| リスク | 対策 |
|--------|------|
| Web収集の安定性 | Google Custom Search APIを使用し、スクレイピングは避ける |
| APIレートリミット | 収集頻度を1時間に1回に制限。キャッシュ活用 |
| 情報の正確性 | 公式ソースを優先。信頼度スコアを付与 |
| デモ当日の動作 | 事前に収集済みデータをキャッシュし、デモ用フォールバックを用意 |

## 差別化ポイント（審査基準へのマッピング）

| 審査基準 | アピールポイント |
